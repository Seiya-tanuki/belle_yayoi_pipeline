# 00_PRD_Overview

## 1. 背景
顧客は「会計インポート用データ生成パイプライン」を求めている。対象はレシート/領収書/請求票/明細（ホテル等）を含む。

## 2. MVPのゴール
1. 画像 → Gemini OCR → JSON（構造化）
2. JSON → 弥生会計インポート用CSV（25項目形式、ヘッダなし）
3. 弥生会計にインポートし、顧客が弥生側で科目の振替等を行える状態にする。

## 3. MVPで「止めない」ための前提
1. OCRは推測禁止。未読は null / unknown と issues で明示する。
2. 変換はAIに頼らず、決定表と算術で行う（監査可能にする）。
3. 危険な入力は FAIL で止め、WARN は出力しつつ人が確認できるようにする（docs/05）。

## 4. MVP範囲外（後続）
1. 勘定科目の自動推定
2. 支払手段（現金/クレカ）判定と貸方科目の自動選択
3. 摘要の高度な整形（品目要約、カテゴリ付与）
